# UI仕様書：FotoNote

## 1. デザイン方針

アプリ全体は、写真を主役にしたシンプルで静かなUIにする。

### キーワード

- シンプル
- 高級感
- 静か
- 写真向け
- カメラらしさ
- エディトリアル
- 余白を活かす
- 派手にしない
- わかりやすい

---

## 2. 画面構成

MVPではメイン画面1つで構成する。

### メインエディタ画面

ユーザーはこの画面で以下を行う。

- 画像選択
- フレーム付き画像のプレビュー
- EXIF情報の確認
- 書き出し
- 広告表示

---

## 4. トップバー

## 4.1 表示要素

- アプリ名：`FotoNote`
- 必要に応じて設定アイコン

## 4.2 挙動

- 画面上部に常時表示する。
- MVPでは複雑なナビゲーションは不要。
- 戻るボタンは基本的に不要。

---

## 5. 空状態

画像が選択されていない場合は、空状態を表示する。

### 表示テキスト

```txt
写真を選択してください
EXIF情報を読み取り、フレーム付き画像を作成します。
```

### ボタン

```txt
画像を選択
```

---

## 6. プレビューエリア

プレビューエリアには、完成予定のフレーム画像を表示する。

### 要件

- 画面幅に収まるように表示する。
- 書き出し画像と同じアスペクト比を維持する。
- プレビュー外の背景は薄いグレーにする。
- EXIF情報が読み込まれたら自動でプレビューに反映する。
- 画像が縦向きでも横向きでも破綻しないようにする。

---

## 7. テンプレート

v0.2では以下の3種類を実装する。

- Classic White
- Soft Gray
- Noir

## 7.1 書き出しキャンバス

```txt
幅1080 px
高さは写真比率とメタデータ表示量に合わせて可変
```

## 7.2 レイアウト

```txt
┌──────────────────────────────┐
│ 白余白                        │
│ ┌──────────────────────────┐ │
│ │                          │ │
│ │ 写真                     │ │
│ │                          │ │
│ └──────────────────────────┘ │
│                              │
│ CAMERA MODEL                 │
│ 24mm  f/1.8  1/250s  ISO100  │
│ 2026.05.10 19:47             │
└──────────────────────────────┘
```

## 7.3 スタイル

- Classic White：白背景、黒系テキスト
- Soft Gray：淡いグレー背景、濃いグレー系テキスト
- Noir：黒背景、白系テキスト
- 写真周囲の余白：48px程度
- 下部メタデータエリア：180pxから240px程度
- 区切り線：任意
- ロゴ画像：使用しない

## 9. 操作エリア

## 9.1 画像選択ボタン

### ラベル

```txt
画像を選択
```

### 挙動

- Androidの画像選択UIを開く。
- ユーザーが1枚の画像を選択する。
- 選択された画像を読み込む。
- EXIF情報を読み取る。
- プレビューを更新する。

---

## 9.2 書き出しボタン

### ラベル

```txt
画像を書き出す
```

### 挙動

- 現在のプレビュー内容をもとに書き出し画像を生成する。
- 生成した画像を端末ギャラリーに保存する。
- 成功時は完了メッセージを表示する。
- 失敗時はエラーメッセージを表示する。

---

## 9.3 テンプレート選択

テンプレートは以下から選択できる。

```txt
Classic White
Soft Gray
Noir
```

---

## 10. 画面下固定広告

## 10.1 配置

- 画面最下部に固定する。
- 操作エリアの下に表示する。
- プレビューや操作ボタンに重ならないようにする。

## 10.2 挙動

- 広告が読み込まれた場合は表示する。
- 広告読み込みに失敗してもアプリUIは崩れないようにする。
- 書き出し画像には広告を含めない。

---

## 11. エラー表示

## 11.1 EXIFが見つからない場合

```txt
EXIF情報が見つかりませんでした。
必要に応じて手動入力機能で補完できます。
```

MVPでは手動入力機能は未実装でもよい。

---

## 11.2 書き出し失敗

```txt
画像の書き出しに失敗しました。
もう一度お試しください。
```

---

## 11.3 保存成功

```txt
画像を保存しました。
```

## 14. アクセシビリティ

- ボタンには明確なラベルを付ける。
- アイコンのみの操作は避ける。
- テキストのコントラストを確保する。
- 画像が未選択の状態でも、次に何をすればよいか分かる表示にする。
