# Codex用プロンプト集

## 使い方

Codexには、フェーズごとに指示を分けて実装させる。

一度にすべて実装させず、以下の順番で進める。

1. プロジェクト作成・画像選択
2. EXIF読み取り
3. フレームプレビュー
4. 画像書き出し
5. 広告実装
6. 調整・リリース準備

---

# 第1段階：プロジェクト作成・画像選択

```txt
このプロジェクトはAndroid向けのKotlin / Jetpack Composeアプリです。

まず、以下のファイルを読んでください。

- AGENTS.md
- docs/PRD.md
- docs/UI_SPEC.md
- docs/TECH_SPEC.md
- docs/TASKS.md

そのうえで、フェーズ1とフェーズ2のみを実装してください。

まだ広告は実装しないでください。
まだ画像書き出しは実装しないでください。
まだEXIF読み取りは実装しないでください。
不要な機能は追加しないでください。

今回のゴールは以下です。

- Androidアプリとして起動できること。
- メインエディタ画面が表示されること。
- ユーザーが画像を1枚選択できること。
- 選択した画像がプレビューエリアに表示されること。
- コードがシンプルで読みやすいこと。
```

---

# 第2段階：EXIF読み取り

```txt
次に、フェーズ3のEXIF読み取りを実装してください。

対象ファイルは以下です。

- docs/TECH_SPEC.md
- docs/TASKS.md

要件は以下です。

- ExifInterfaceを追加する。
- ExifDataモデルを作成する。
- ExifReaderクラスを作成する。
- 選択画像からEXIF情報を読み取る。
- 読み取ったEXIF情報を画面に表示する。
- EXIFが存在しない画像でもクラッシュしないようにする。
- GPS情報は読み取っても表示しないでください。
- UIはまだ大きく変更しないでください。
```

---

# 第3段階：Classic Whiteテンプレート

```txt
次に、フェーズ4とフェーズ5を実装してください。

目的は、Classic Whiteテンプレートのプレビューを作ることです。

要件は以下です。

- 白背景のフレームを作成する。
- 選択画像をフレーム内に表示する。
- 写真下部にEXIF情報を表示する。
- 表示項目は以下にしてください。
  - カメラメーカー
  - カメラ機種名
  - 焦点距離
  - F値
  - シャッタースピード
  - ISO
  - 撮影日時
- 書き出しサイズは幅1080px、高さは写真比率に合わせる想定にしてください。
- プレビューでは画面幅に合わせて縮小表示してください。
- 縦画像・横画像の両方で崩れないようにしてください。
- 実在ブランドのロゴ画像は使用しないでください。
```

---

# 第4段階：画像書き出し

```txt
次に、フェーズ6の画像書き出しを実装してください。

要件は以下です。

- 現在のテンプレート内容をBitmapとして生成する。
- 書き出しサイズは幅1080px、高さは写真比率に合わせて可変にしてください。
- 背景は白にしてください。
- 写真を指定エリアに描画してください。
- 写真下部にEXIF情報を描画してください。
- 書き出し画像にはUIボタンを含めないでください。
- 書き出し画像には広告を含めないでください。
- JPEG形式、品質95で保存してください。
- MediaStoreを使ってギャラリーに保存してください。
- 保存成功時にメッセージを表示してください。
- 保存失敗時にエラーメッセージを表示してください。
```

---

# 第5段階：広告実装

```txt
次に、フェーズ7の広告実装を行ってください。

要件は以下です。

- Google Mobile Ads SDKを追加してください。
- Mobile Adsを初期化してください。
- 画面下に固定バナー広告を表示してください。
- 開発中はAdMobのテスト広告ユニットIDを使用してください。
- 広告は操作ボタンと重ならないようにしてください。
- 広告読み込みに失敗してもアプリがクラッシュしないようにしてください。
- 書き出し画像に広告が含まれないことを維持してください。
- インタースティシャル広告は追加しないでください。
- リワード広告は追加しないでください。
- アプリ内課金は追加しないでください。
```

---

# 第6段階：調整

```txt
次に、フェーズ8の調整を行ってください。

要件は以下です。

- 余白を調整してください。
- 文字サイズを調整してください。
- メタデータの表示順を整理してください。
- 書き出し中のローディング表示を追加してください。
- 日本語UIラベルを整理してください。
- 複数の画面サイズで崩れにくいレイアウトにしてください。
- ダークモードでも見づらくならないようにしてください。
- 既存機能を壊さないでください。
```

---

# 第7段階：リリース前確認

```txt
リリース前の確認を行ってください。

以下を確認してください。

- テスト広告IDが本番広告IDに差し替え可能な構成になっていること。
- プライバシーポリシーが必要な構成になっていること。
- 不要な権限を要求していないこと。
- 画像がサーバーに送信されていないこと。
- EXIF情報がサーバーに送信されていないこと。
- GPS情報が表示されていないこと。
- EXIFがない画像でもクラッシュしないこと。
- 書き出し画像に広告が含まれていないこと。
- 書き出し画像にUIボタンが含まれていないこと。
- リリースビルドが作成できること。
```
